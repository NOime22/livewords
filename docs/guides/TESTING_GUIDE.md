# LiveWords Testing Guide

Architecture source of truth: `docs/CURRENT_ARCHITECTURE.md`.

## 1. Command Matrix

Run from repository root:

- `npm run verify` - baseline lint and tests
- `npm run smoke:userData` - user data contract smoke checks
- `npm run smoke:storyData` - story contract smoke checks
- `npm run smoke:remote:userData` - remote cloud smoke for userData
- `npm run smoke:remote:storyData` - remote cloud smoke for storyData
- `npm run smoke:remote` - aggregated remote smoke checks
- `npm run smoke:db-index` - critical index/explain static checks
- `npm run verify:rollout` - rollout playbook structure validation
- `npm run verify:regression` - unified release regression pack

Remote smoke requires environment variables:
- `CLOUDBASE_ENV_ID`
- `CLOUDBASE_SECRET_ID` (or `TENCENTCLOUD_SECRETID`)
- `CLOUDBASE_SECRET_KEY` (or `TENCENTCLOUD_SECRETKEY`)
- If using temporary credentials: `CLOUDBASE_SESSION_TOKEN` (or `TENCENTCLOUD_SESSIONTOKEN`)

## 2. Must-Pass Invariant Scenarios

### 2.1 `operationId` idempotency

- Replaying same mutation payload with same `operationId` must dedupe.
- Replaying with different `operationId` must apply independent mutation.

### 2.2 `expectedRev` optimistic concurrency

- Story write with matching `expectedRev` succeeds and increments revision.
- Stale `expectedRev` returns `REV_CONFLICT` without mutation.

### 2.3 Expiry retain-and-mark and revival bound

- Expired story remains in `users.activeStory` with `status: 'expired'`.
- Eligibility relies on `reviveEligibleUntil` and `reviveCount`.
- Over-limit revival attempt fails deterministically.

### 2.4 Protagonist prompt mode metadata

- Prompt metadata exposes protagonist source and enabled context flags.
- Sanitized/fallback protagonist behavior is deterministic.

### 2.5 Mid-week branch immutability

- First valid branch choice persists and drives later prompt context.
- Conflicting second choice returns `BRANCH_IMMUTABLE`.

## 3. Release Verification Flow

1. Run `npm run verify`.
2. Run `npm run smoke:userData && npm run smoke:storyData`.
3. Run `npm run verify:rollout`.
4. Run `npm run verify:regression`.
5. For staging/prod promotion, run `npm run smoke:remote`.

All steps must exit zero before promotion.

## 4. Evidence Convention

- Evidence artifacts live in `.sisyphus/evidence/`.
- Use `task-{N}-{scenario}.{txt|json}` naming.
- Regression evidence index is regenerated by `npm run verify:regression`.
